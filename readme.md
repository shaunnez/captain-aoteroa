
# B1 My Funding NZ Disability Budget Manager
What exists in NZ
No NZ product. The new flexible funding model launches April 2026. Geneva Healthcare's iThrive helps with some funding flexibility. Mycare handles payments. But no dedicated budget tracking and planning tool exists for the NZ system.
Real gap
Large. The system is about to change and there's no tooling ready for it. Direct parallel to Australia's NDIS ecosystem of budget apps (Leap in!, Self Manager, Peasy).
Build feasibility
High. Standard financial tracking web app. Needs to integrate with NASC allocation data (format TBD as the system is still being designed).
Effort estimate
8 to 12 weeks for MVP. Will need iteration as the funding model details finalise.
Verdict
TOP PICK. Timing is perfect: the system launches in 2026, no tooling exists, and Australia has proven the model works. Risk: government may build their own (but historically they don't build good UX). First mover advantage is real.

# B9 FormBridge: AI Government Form Assistant
What exists in NZ
No NZ product. Government forms are available online but not accessible for people with motor impairments or cognitive fatigue. Some community organisations help people fill out forms manually.
Real gap
Large. The forms are a documented barrier and no tech solution exists. This is universally hated by disabled people and their advocates.
Build feasibility
High. Conversational form interfaces are well-understood. Voice input, save-and-resume, and PDF output are standard. The AI layer for jargon explanation and pre-filling is straightforward with current LLMs.
Effort estimate
6 to 8 weeks for first 3 to 4 forms. Ongoing work to add more forms.
Verdict
TOP PICK. Universal pain point, clear gap, buildable in the Access Activator timeframe, and directly addresses the digital services priority. Start with the 3 most-applied-for forms: Disability Allowance, Supported Living Payment, and Total Mobility. Demo a real person completing a form by voice during the showcase.

# A6 Caption Aotearoa: Community Event Captioning
What exists in NZ
No NZ product for community-level live captioning. Professional captioning services exist but are expensive ($150+/hour). Google Live Transcribe works for individuals but not event broadcast. Kara Technologies is building NZSL avatar tech but it's not available as a captioning service yet.
Real gap
Large. The gap between expensive professional captioning and free-but-personal phone apps is completely unserved for community events.
Build feasibility
High. AI transcription APIs (Whisper, Google Speech, Azure) are mature. The tech is a web app that connects audio to transcription and broadcasts captions to audience phones. The hard part is accuracy for NZ accents and te reo terms.
Effort estimate
4 to 6 weeks for MVP. Browser-based, no app install needed.
Verdict
TOP PICK. Low build effort, high impact, no existing NZ competition, works for both Deaf and hard-of-hearing communities. Perfect for Access Activator demo: caption an actual community event live during the showcase. Elegant story.

# A5 Kaitiaki: Accessible Transport Companion
What exists in NZ
Auckland Transport app exists but has poor accessibility. AT's GTFS feed is public. Blind Low Vision NZ advocates for transport accessibility. No NZ app provides audio bus announcements, haptic alerts, or footpath-safe routing for blind users.
Real gap
Large. The public transport data is available but no one has built an accessible interface on top of it for blind and visually impaired users.
Build feasibility
High. GTFS real-time feeds are public and well-documented. Audio/haptic notification is standard mobile dev. Footpath routing is harder (needs council data on construction, shared spaces).
Effort estimate
6 to 8 weeks for core bus tracking with audio. 3 to 4 months for full routing.
Verdict
TOP PICK. Clear gap, public data available, high daily impact, strong transport priority alignment with Whaikaha. Could demo a working prototype within the Access Activator timeframe. Start Auckland-only.

# B3 AccessRoute Aotearoa: Wheelchair Navigation
What exists in NZ
Google Maps has basic wheelchair routing. Wheelmap is sparse in NZ. No NZ-specific wheelchair pedestrian routing using council footpath data exists.
Real gap
Medium to large. The data exists in council GIS systems but isn't accessible to end users in a useful format.
Build feasibility
Medium. Ingesting and normalising council GIS data across multiple different council systems is messy but doable. GTFS integration is straightforward.
Effort estimate
3 to 4 months for Auckland. 6+ months for multi-city.
Verdict
Strong idea, good grant fit, but the council data integration is the unglamorous hard work. Start with Auckland Council's open data portal. Could pair well with A5 (Kaitiaki) as a shared transport accessibility platform.
